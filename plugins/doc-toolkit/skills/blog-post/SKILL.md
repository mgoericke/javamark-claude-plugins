# Blog-Post Skill

## Purpose
Create blog posts that present technical projects in an understandable and entertaining way.

## Style
- Personal yet technically sound
- Code examples embedded and explained
- Architecture decisions motivated
- Lessons learned highlighted
- English

## Structure
1. **Hook**: Why is this topic relevant?
2. **Problem**: What did we want to solve?
3. **Approach**: How did we go about it?
4. **Implementation**: Core code with explanations
5. **Demo**: What can you see/try out?
6. **Conclusion**: What did we learn?

## Notes
- "Get a foot in the door" — concise rather than exhaustive
- Blog-worthy demos: Entertaining, educational, easy to follow
- Include architecture diagrams as Mermaid
